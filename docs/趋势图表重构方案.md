# 趋势图表重构方案：从折线图到离散气泡图

## 一、 核心设计理念转变

- **过去（折线图逻辑）**：强调“连续性”。一条线代表一个事件的完整生命周期，强制连接相邻时间点。痛点是事件掉出榜单后线条断裂、颜色复用混乱、多事件交叉成蜘蛛网。
- **现在（离散图逻辑）**：强调“状态切片”。每次数据抓取（如每5分钟）都是一次独立的快照。榜单上有什么事件，就在图上画什么点。掉出榜单即无数据，新进榜单即新增点。**彻底消灭连线的视觉负担。**

## 二、 坐标系与数据维度定义（从二维到多维）

摒弃单纯的X-Y轴，采用**“时间-排名离散气泡图”**，利用点的属性承载更多信息：

- **X轴（横轴）**：时间轴。今日视图按分钟/小时计；7天视图按天计。

- **Y轴（纵轴）**：榜单排名。**关键操作：反转Y轴**，使Rank 1在最顶端，Rank 5在底部，符合人类阅读直觉。

- 数据点形态（气泡）

  ：

  - **颜色**：代表事件ID（A、B、C等），或直接代表情绪属性（红色=负面争议，绿色=正面，灰色=中性）。
  - **大小（半径）**：绑定该热点的“平均热度值”（如截图中的99,196）。热度越高，圆点越大。
  - **透明度**：设置为基础透明度（如60%），当多个事件在同一时间段密集更迭时，重叠的圆点会自然加深颜色，形成“密度热力感”。

## 三、 核心场景交互规则（解决ABCDE进出问题）

### 1. 事件掉出榜单（如A从第1掉到第5，再掉出前五）

- **表现**：A的圆点会随时间向右下方滑动至Rank 5。到达下一个抓取时间点时，如果A已掉出前五，图表上**直接不再绘制A的圆点**。
- **视觉优化**：在A最后一个存在的圆点（Rank 5）处，可添加一个微小的“渐隐”或“向右淡出”的动效，暗示该热点已退场，无需画任何断线标记。

### 2. 事件升入榜单（如E从前六升入前五）

- **表现**：在E未进入前五的时间段，图表上没有E的痕迹。当E升入（如Rank 4）时，在对应的时间点**直接生成一个新的圆点**，并向右延伸。
- **视觉优化**：在E新出现的那个圆点外围，添加一个“脉冲高亮”或“水波纹”动效，并在旁边显示极小的“NEW”标签，吸引用户注意新晋热点。

### 3. 悬停交互——按需还原事件轨迹（至关重要）

- **默认状态**：图上只有干干净净的散点，互不干扰，展示宏观的热点密度和排名更迭。
- **悬停状态**：当用户鼠标悬停在事件A的任意一个圆点上时，**前端动态拉出一条半透明的虚线**，将该时刻之前和之后所有属于A的圆点串联起来，并高亮，其他事件的圆点变暗。
- **价值**：既保证了默认界面的清爽，又保留了折线图“追踪单个事件发展轨迹”的能力。

## 四、 模块级落地方案（配合“可接受删功能”原则）

### 模块A：“今日”实时榜单 -> 替换为 实时离散气泡图

- **操作**：直接用上述设计的“时间-排名离散气泡图”替换原有的今日折线图。
- **效果**：完美呈现今日5个位置的实时更迭。圆点大小直观展示热度爆发，颜色展示情绪，彻底解决A出E进的连线错乱。

### 模块B：“最近7天”趋势 -> 降维重构为 宏观聚合散点/密度图

- **痛点回顾**：7天内上百个热点轮换，如果每个都画散点，依然是满屏麻子，不可读。

- 重构方案

  ：放弃“单事件7天轨迹”功能，将其重构为

  宏观热点密度散点图

  。

  - X轴为7天，Y轴为排名1-5。
  - 只画点，不追踪事件ID。
  - 点的颜色不再代表具体事件，而是代表**该时间段榜单的整体情绪占比**（如偏红代表负面争议集中，偏绿代表正面）。
  - 点的大小代表**该时间段的平均热度峰值**。

- **效果**：将原本混乱的“单事件折线图”降维成一张“7天舆情大盘热力图”。用户一眼就能看出“最近7天哪几天热点最密集、哪几天负面情绪最重”，保住了功能且大幅提升了专业度。

## 五、 前端实施建议（以ECharts为例）

1. **图表类型**：使用 `type: 'scatter'`。

2. 数据处理

   ：

   - 后端只需返回扁平化数组：`[时间戳, 排名, 事件名, 热度值, 情绪类型]`。
   - 无需像折线图那样为了补全断开区间去填充 `null`，离散图天然支持数据长度不一致。

3. 关键配置项

   ：

   - `yAxis: { inverse: true, min: 1, max: 5 }` （反转Y轴，锁定1-5名）。
   - `series.symbolSize: function(data) { return Math.sqrt(data[3]) / 10; }` （根据热度值动态计算圆点大小）。
   - `emphasis` 配合 `focus: 'series'` 实现悬停高亮单个事件所有散点的效果。

## 总结

这份设计思路通过**“去连线化”**解决了您的核心痛点，通过**“气泡多维化”**增加了数据承载量，最后通过**“悬停连线”**保留了追踪能力。同时将7天功能从“微观追踪”降级为“宏观密度”，既满足了删减冗余功能的需求，又让整个网页焕发出现代数据看板的高级感。
